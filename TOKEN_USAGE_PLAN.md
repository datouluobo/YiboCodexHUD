# Token 用量显示开发方案

## 目标

新增一个和现有 HUD 显示项平级的 `Token 用量` 项。

`Token 用量` 只负责 token 相关统计，不引入趋势图、费用估算、模型统计或其它非必要功能。它需要参与现有 HUD 的显示开关和顺序调整，和以下项目处于同一层级：

- 短周期
- 长周期
- Token 用量
- 重置次数

## HUD 显示内容

Token 用量支持三个统计范围：

### 本次

显示当前会话或当前任务内的 token 用量。

示例：

```text
本次 42k
```

### 今日

显示当天累计 token 用量。

示例：

```text
今日 128k
```

### 本周期

显示从最近一次额度周期起点开始累计的 token 用量。

如果官方额度自动重置，或用户使用重置次数导致额度窗口重新计算，本周期重新开始计算。

示例：

```text
本周期 380k
```

## 显示格式

每个统计范围支持两种格式。

### 总量

只显示输入和输出 token 的合计值。

示例：

```text
今日 128k
```

### 输入 / 输出拆分

分别显示输入 token 和输出 token。

示例：

```text
今日 入 90k / 出 38k
```

如果多个范围同时开启，HUD 内部组合显示。

总量模式示例：

```text
本次 42k · 今日 128k · 本周期 380k
```

拆分模式示例：

```text
本次 入 30k / 出 12k · 今日 入 90k / 出 38k
```

## 设置项

在设置页的显示内容区域新增 `Token 用量` 分组。

### 一级开关

- 显示 Token 用量

### 范围开关

- 显示本次
- 显示今日
- 显示本周期

至少保留一个范围开启。若用户关闭最后一个范围，保留当前最后一项或自动回退到 `显示今日`。

### 格式切换

- 总量
- 输入 / 输出拆分

该切换作用于所有已开启的 Token 范围。

### 单位格式

- 自动简写：`950`、`12.4k`、`1.2M`
- 完整数字：`128432`

第一版默认使用自动简写。

## 显示顺序

在现有显示顺序设置里新增 `Token 用量`。

排序列表调整为：

- 短周期
- 长周期
- Token 用量
- 重置次数

用户可以通过上移 / 下移调整它们在 HUD 中的展示顺序。

## 本周期计算规则

本周期统计需要识别周期边界。

满足以下任一条件时，视为进入新周期：

- 短周期或长周期的 `resetsAt` 发生跳变
- 用户使用重置次数后，额度窗口重新计算
- 官方返回的额度窗口表现出已重置状态

进入新周期后：

- 本周期输入 token 清零
- 本周期输出 token 清零
- 更新当前周期起点
- 记录新的短周期和长周期重置时间
- 记录新的重置次数状态

如果无法可靠判断周期变化，则保留上次周期起点，不主动清零。

## 本地持久化数据

为了支持本周期统计，需要在本地保存最小状态。

建议保存：

- 当前周期起点
- 本周期输入 token
- 本周期输出 token
- 上次看到的短周期重置时间
- 上次看到的长周期重置时间
- 上次看到的重置次数
- 上次成功刷新时间

这些状态只服务于 token 周期累计，不影响现有额度显示。

## 数据缺失处理

如果 token 数据不可用：

- 隐藏 Token 用量项
- 保留短周期、长周期、重置次数等已有显示项
- 不显示错误文本挤占 HUD
- 可在日志里记录失败原因

如果部分范围不可用：

- 只隐藏不可用范围
- 其它可用范围继续显示

例如 `本次` 不可用但 `今日` 可用时：

```text
今日 128k
```

## 第一版范围

第一版只实现：

- Token 用量作为独立 HUD 显示项
- 支持本次 / 今日 / 本周期
- 支持总量和输入 / 输出拆分切换
- 支持自动简写单位
- 支持和短周期 / 长周期 / 重置次数一起排序
- 数据缺失时隐藏 Token 项

第一版暂不实现：

- 趋势图
- 费用估算
- 模型维度统计
- 历史详情页
- token 阈值告警
- 导出分析

## 推荐默认值

- 显示 Token 用量：开启
- 显示本次：开启
- 显示今日：开启
- 显示本周期：开启
- 显示格式：总量
- 单位格式：自动简写
- 默认顺序：短周期、长周期、Token 用量、重置次数
